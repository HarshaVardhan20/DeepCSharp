1. Introduction

In real-world software, data inside a class should never be exposed directly. C# provides Properties and Indexers to safely access and manipulate internal data. 

User → Property / Indexer → Internal Data
Properties and Indexers are fundamental to writing secure, maintainable, and professional C# code.

2. Fields vs Properties

❌ Public Fields (Bad Practice)

public int Age;

✅ Properties (Recommended)

private int age;
public int Age
{
    get { return age; }
    set { age = value; }
}
Public FieldProperty
No validationSupports validation
No controlAccess control possible
Bad designIndustry standard
Never expose class data directly using public fields.

3. Property Syntax (Step-by-Step)

class Employee
{
    private double salary;

    public double Salary
    {
        get
        {
            return salary;
        }
        set
        {
            salary = value;
        }
    }
}

• get → returns the value • set → assigns the value

Think of properties as “smart fields”.

4. Validation Using Properties

public int Age
{
    get { return age; }
    set
    {
        if(value < 0)
            throw new Exception("Age cannot be negative");
        age = value;
    }
}
Real-World Use: Banking systems validating balance, age, credit limits.

5. Property Accessors

public int Id
{
    get;
    private set;
}

• Property can be read-only • Setter can be restricted

Very common in Entity and DTO classes.

6. Read-only & Write-only Properties

public int EmployeeId { get; }
public string Password { private get; set; }
Real-World: Employee ID should not change. Passwords should never be read.

7. What are Indexers?

Indexers allow objects to behave like arrays. They enable accessing class data using index syntax. 

object[index]
ConfigurationManager["AppName"]

8. Indexer Syntax

class StringCollection
{
    private string[] data = new string[5];

    public string this[int index]
    {
        get { return data[index]; }
        set { data[index] = value; }
    }
}
Indexer always uses this keyword.

9. Real-World Indexer Example

class Settings
{
    private Dictionary settings =
        new Dictionary();

    public string this[string key]
    {
        get { return settings[key]; }
        set { settings[key] = value; }
    }
}
settings["Theme"] = "Dark";

  • Always use properties for fields
  • Use validation inside setters
  • Keep accessors lightweight
  • Indexers should be intuitive
