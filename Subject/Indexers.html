Introduction to Properties

Properties are members that expose class data safely. They combine the flexibility of methods with the simplicity of fields. Properties support validation, encapsulation, and maintainability. 

Class
 └── Private Field
      └── Property (get / set)
Real-World Use: Employee Salary validation before assignment.
Beginner Tip: Always expose fields using properties.
❮ Previous Next ❯

Why Properties Are Needed

Direct access to fields breaks encapsulation. Properties allow validation, logging, and access control without changing the public API. 

Bad Design → public fields
Good Design → private fields + properties
Warning: Public fields expose internal state.

Property Syntax

class Person
{
    private int age;

    public int Age
    {
        get { return age; }
        set { age = value; }
    }
}
Explanation: get returns value, set assigns value.

Property Accessors

Accessors define how a property is read or written. You can restrict access using different access modifiers. 

public int Age
{
    get;
    private set;
}
Advanced Note: Use private set for immutable objects.

Read-only and Write-only Properties

public int Id { get; }
public string Password { private get; set; }
Real-World Use: Read-only IDs, write-only passwords.

Introduction to Indexers

Indexers enable indexed access to class data. They improve readability and usability of custom collections. 

object[index]
Real-World Use: Custom cache, configuration store.
❮ Previous Next ❯

Indexer Syntax

class DataStore
{
    private string[] data = new string[5];

    public string this[int index]
    {
        get { return data[index]; }
        set { data[index] = value; }
    }
}
Indexers use the this keyword.
