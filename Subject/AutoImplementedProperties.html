C# Auto-Implemented Properties

Auto-implemented properties provide a concise syntax for property declarations when no additional logic (such as validation or notification) is required in the property accessors.

When you define an auto-implemented property, the compiler automatically creates a private, anonymous backing field for the property.

Syntax

The syntax is straightforward. You simply declare the property with get; and set; keywords without a body for the accessors.

Example

The general syntax for an auto-implemented property:

public DataType PropertyName { get; set; }

Tip:

This syntax significantly reduces the boilerplate code required for simple properties, making your code cleaner and easier to read.

When to Use

Auto-implemented properties are best used when:

  • You need to store data but don't need validation logic in the set accessor.
  • You don't need any computed values in the get accessor.
  • You don't need to access the backing field directly from within the class (as it's hidden).

Note:

If you need to add logic later (e.g., logging when a value changes), you can convert an auto-implemented property to a regular property with a defined backing field and accessors without breaking compatibility for users of the property.

Example: Basic Usage

A simple class using two auto-implemented properties:

Example Code:

using System;

class Person
{
    // Auto-implemented properties
    public string Name { get; set; }
    public int Age { get; set; }
}

class Program
{
    static void Main()
    {
        Person person1 = new Person();
        person1.Name = "Alice";
        person1.Age = 30;
        Console.WriteLine($"Name: {person1.Name}, Age: {person1.Age}");
    }
}

Example: Read-Only Property

You can make an auto-implemented property read-only by omitting the set; accessor.

The value must be set in the constructor of the class.

Example Code:

class ImmutablePoint
{
    // Read-only auto-property
    public int X { get; }
    public int Y { get; }

    // Constructor to set initial values
    public ImmutablePoint(int x, int y)
    {
        X = x;
        Y = y;
    }
}

Example: Private Setter

You can restrict the accessibility of the set; accessor to within the class itself, while keeping the get; accessor public.

Example Code:

using System;

class GameScore
{
    // Public getter, private setter
    public int Score { get; private set; }

    public GameScore()
    {
        Score = 0; // OK: within the class
    }

    public void AddPoints(int points)
    {
        Score += points; // OK: within the class
    }
}

class Program
{
    static void Main()
    {
        GameScore game = new GameScore();
        game.AddPoints(100);
        Console.WriteLine($"Score: {game.Score}");
        // game.Score = 50; // ERROR: Cannot set from outside the class
    }
}
